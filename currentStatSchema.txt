1) App features table
Input events used

ForegroundAppChanged(appKey, category)
AppDwell(appKey, category, durationMs, reason?)
Table: app_window_features


Core columns (recommended v1)

app_switch_count = count of ForegroundAppChanged in the window

app_unique_count = distinct appKey with dwell overlap > 0 in window

app_dwell_mean_ms = mean of dwell segment durations overlapping window

app_dwell_std_ms

app_dwell_max_ms

app_top1_share = max_app_dwell_ms / max(app_dwell_total_ms, 1)

Category ratio columns (stable schema)
cat_browser_ratio,cat_ide_ratio,cat_comms_ratio,cat_other_ratio (expand later)

Each = category dwell overlap / (window_sec * 1000)

Data quality

has_app_data (0/1)

How to compute dwell overlap (the key detail)

Each AppDwell event gives a segment:

seg_end = event.ts
seg_start = event.ts - durationMs
That segment may cross multiple windows → split overlap:

For each window it overlaps:
overlap_ms = max(0, min(seg_end, win_end) - max(seg_start, win_start))
add overlap_ms to that appKey/category bucket

2) Session features table
Input events used

IdleSample(idleMs, idleBucketSec)
SessionLock(reason)
SessionUnlock(reason)
DisplayOn/DisplayOff(displayState, source)
ScreenSaverOn/Off(running, idleBucketSec, idleMs) (you currently have Off; schema can include both)

Table: session_window_features

Core columns

lock_count = count(SessionLock in window)
State ratios (integrated over time)
Compute by holding the last known state until next event:

locked_ratio = time locked / window duration
display_off_ratio = time display == Off / window duration
screensaver_on_ratio = time screensaver running / window duration (0 if never observed)

Idle stats (time-weighted)
From idleBucketSec state:

idle_bucket_mean_sec = weighted mean over time
idle_bucket_max_sec = max seen (state-wise)
idle_ge_60_ratio = fraction time idleBucketSec >= 60

attribute that interval’s duration to the windows it overlaps
add to locked_time_ms, display_off_time_ms, etc.
